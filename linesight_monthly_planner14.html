Capacity basis default setup min

Quantities are booked on the batch start day and automatically spill over into the following idle days (multi-day batches). Green = fits in one day · blue = multi-day batch (↳ marks continuation days with the quantity produced there) · red = load that cannot be absorbed before month-end · yellow = batch starting on a weekend/holiday. Cells show the full planned quantity; hover a cell for the carry-over details. The capacity basis (top right) decides how a day fills up: Units/day uses each line's flat u/h rate, Std. minutes/day uses each SKU's standard minutes per unit plus setup time, and Run time/batch uses each batch's actual run minutes (imported from HC_Total_Min / HC_Total_Hrs), the most reliable option when your sheet already carries it. Each batch is its own chip — drag a chip onto another day or line to reschedule it, or into the un-allocated tray below to park it. Completed batches (from the Status/Produced columns) show struck-through with a ✓; the Adherence tile tracks produced vs. planned.

Opening backlog — by product group

Opening backlog = unfulfilled demand carried into this month from earlier periods, per product group. Any MPS spill-over (batches the previous month could not produce) is added automatically on top of the manual value.
Weekly backlog = max(0, previous backlog + demand − capacity). Free capacity in a week works the backlog down. The value next to each input shows that group's projected month-end backlog.

Backlog by line — month end

Each product group's opening backlog is allocated to its own lines by demand share. Each line's backlog rolls week to week against its own capacity.
